(a.k.a. ReadMe.htm)

 

 

Package files:

 

This package must contains:

 

CBlaster.exe

CharBlaster executable.

1046.cbl

CharBlaster pt-br language file.

VBlaster.exe

VaultBlaster executable.

1046.vbl

VaultBlaster pt-br language file.

MuData.lib

MuOnline data library. This file contains some data such item names, codes, spot locations, etc.

Src.cab

Cabinet with source codes of VaultBlaster, CharBlaster and BCompiler. There is some stuff too. They aren't documented, then if you are programmer, start working now, you have a lot of work to do ^_^. Any questions, mail me.

ReadMe.htm

LeiaMe.htm

This file and its version in Brazillian Portuguese.

 

Note:

CharBlaster and VaultBlaster also create files named blaster.cws and blaster.vws, respectively. It's a Workspace save; don't worry about it, even if you get the two deleted. The programs will create two others =P.

 

Version 1.1 new!

 

The version 1.1 also comes with a folder called BCompiler, which contains a compiler for MuData.lib. In this folder you'll find four files:

 

BCompiler.exe

Blaster Compiler. By default creates a file named MuData.lib, but you can choose it.

DefCompile.bat

Command-line arguments for default compilation. In the most cases, every time you edit data.src you may use this batch to compile it.

Data.src

Source plain text file for mudata.lib contained in this package.

Datasrc.htm

File with information about Data.src format, for your own editing.

 

You can modify Data.src for your own purposes (e.g.: to get the items from newer versions of MuOnline) and recompile it, replacing the original in package root.

 

Read BCompiler\Datasrc.htm for more information about editing Data.src.

 

Note:

Be sure that you made backup before replacing.

 

FAQ (Frequently Asked Questions):

 

This section is very very necessary because many people had come to me asking the same few questions.

 

 

Q: "Hey man, why I can't blast the server where I play?"

 

A: Dunno, it depends on what error are you getting.

 

1- If are you getting a time-out error, you should verify if the server where are you playing doesn't have a firewall. If yes, forget blasting...

 

2- If no, Verify if the PC that are you using doesn't have a firewall, proxy or modem setup blocking the port that you want to use. If yes, try to reconfigure it. I WON'T HELP YOU ON THIS WAY. FIND IT BY YOURSELF.

 

3- If no, be sure that you are using the correct DATAPORT on the field 'Dataport'. Note that DATAPORT doesn't mean GamePort (44405). Data port usually is 55960. If there is no firewall in the server, you can run a portscan to detect which port is the DataPort. Note, on more time, that I WON'T HELP YOU ON THIS WAY. My purpose isn’t to teach people to run a portscan.

 

4- If the error that you are getting is "No Route to Host", dunno either. This error seems to be linked with ADSL modems setup, but I'm not sure about that. If you get this error, try to run another programs to connect on the server dataport (e.g. a FTP client), just to check.

 

YOU ALSO MUST TO KNOW IT

 

In the VaultBlaster, the field labeled 'UserID' means your username (the one you put in the game's main screen). The Vault is user-associed. But in the CharBlaster, the field labeled 'Name', in 'CharInfo' means your char name. Don't forget it.
