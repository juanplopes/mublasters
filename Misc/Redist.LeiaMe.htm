Arquivos:

 

Este pacote deve conter os arquivos:

 

CBlaster.exe

Executável do CharBlaster.

1046.cbl

Tradução do CharBlaster para o português do Brasil (o programa foi originalmente feito em inglês).

VBlaster.exe

Executável do VaultBlaster.

1046.vbl

Tradução do VaultBlaster para o português do Brasil.

MuData.lib

Biblioteca de dados do MuOnline. Contém algumas informações como nomes de itens, códigos, localizações, etc.

Src.cab

Arquivo CAB com os códigos-fonte do VaultBlaster, CharBlaster e BCompiler. Tem também vários arquivos úteis. Mas atenção: não está documentado, portanto se você é programador, comece a trabalhar agora, você tem muito trabalho pela frente. Qualquer dúvida, mande-me um email.

LeiaMe.htm

ReadMe.htm

Este arquivo e sua versão em inglês.

 

Observação:

O CharBlaster e o VaultBlaster também criam dois arquivos quando são executados, chamados, respectivamente, blaster.cws e blaster.vws. São apenas dados voláteis do programa, você não precisa se preocupar com eles, mesmo se deletá-los por engano. Os programas irão criar outros dois quando forem executados.

 

Novidade da versão 1.1!

 

A versão 1.1 agora vem com uma pasta chamada BCompiler, que contém um compilador para o MuData.lib. Nessa pasta você vai encontrar quatro arquivos.

 

BCompiler.exe

Blaster Compiler. É o executável do compilador. Cria por padrão o arquivo chamado MuData.lib, mas você pode configurar isso.

DefCompile.bat

Arquivo batch pra compilação padrão. O compilador pode ser executado sem parâmetros, mas esse batch ajuda a automatizar o processo.

Data.src

Arquivo-fonte para o MuData.lib que veio nesse pacote.

Datasrc.htm

Documento com informações sobre o formato do Data.src, pra que você mesmo possa editar. O documento está em inglês.

 

Você pode modificar o Data.src para os seus próprios propósitos (e.g.: para conseguir itens de novas versões do MU) e recompilá-lo, substituindo o MuData.lib original da raiz do pacote.

 

Leia o BCompiler\Datasrc.htm pra mais informações sobre esse tipo de edição.

 

Observação:

É importante fazer um backup do MuData.lib antes de realizar a edição.

 

FAQ (Perguntas Freqüentes):

 

Essa seção é realmente necessária, porque muitas pessoas vêm a mim perguntando sempre as mesmas perguntas.

 

P: "Ow, manow, num consigu editar o bau/char, pq?"

 

R: Não sei, depende do erro que está ocorrendo.

 

1- Se está ocorrendo um erro de 'time-out', você precisa verificar se o server que você está tentando editar não tem um firewall. Se tiver, esquece, não dá pra editar.

 

2- Se não, verifique se VOCÊ não está usando um firewall ou proxy que o impeça de conectar usando determinadas portas. Verifique ainda se a configuração do seu modem não impede isso. EU NÃO VOU LHE AJUDAR NESSA CONFIGURAÇÃO, DESCUBRA POR VOCÊ MESMO.

  

3- Se este ainda não for o problema, verifique se você está usando a DataPort correta. Note que DataPort não é a GamePort (44405). Data port normalmente é 55960. Se não tiver firewall no server, você poderá rodar um portscan para detectar qual porta é. Ratifico que NÃO VOU LHE AJUDAR NESSE SENTIDO. Meu propósito não é ensinar às pessoas a usarem um portscan.

 

4- Se o erro que você está tendo for algo como "No Route to Host", realmente não sei como lhe ajudar. Esse erro é específico da camada Socket da rede, não tem a ver com o meu programa. Parece que está relacionado com modems ADSL, mas eu não tenho certeza. De qualquer forma, tente usar qualquer outro programa pra conectar na mesma porta (como um FTP, por exemplo). Só pra verificar.

 

VOCÊ TAMBÉM PRECISA SABER DISSO

 

No VaultBlaster, o campo 'UserID' é destinado ao seu login (aquele que você coloca na tela principal do jogo). O Vault é associado ao login, não ao char. Já no CharBlaster, o campo 'Name' em 'CharInfo' é destinado ao nome do seu char. Não esqueça isto.
